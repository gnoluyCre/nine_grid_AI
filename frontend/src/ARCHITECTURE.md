# frontend/src 目录说明

本目录是前端主源码区，负责页面、组件、状态、类型和 API 调用。  
应用从 `main.tsx` 启动，经 `App.tsx` 组装路由，再分发到各页面与组件。  
主要子目录为 `components`、`pages`、`lib`、`context`、`types`。

| 文件名 | 地位 | 功能 |
| --- | --- | --- |
| `App.tsx` | 路由壳 | 组织前端页面路由 |
| `main.tsx` | 启动入口 | 挂载 React 应用与 Provider |
| `styles.css` | 全局样式 | 定义基础视觉与通用样式类 |
| `vite-env.d.ts` | 声明文件 | 提供 Vite 环境类型声明 |

子目录提示：`components` 放通用 UI，`pages` 放页面容器，`lib` 放工具与接口，`context` 放全局状态，`types` 放共享类型。  
其中 `AuthContext.tsx` 负责登录后的二次校验，`UserAccountPanel.tsx` 负责登录态展示、昵称编辑与安全退出，`RecordsPage.tsx` 和 `InputPage.tsx` 依赖鉴权状态决定是否请求受保护接口。  
`BirthForm.tsx` 与 `SubmitAction.tsx` 共同控制“开始排盘”按钮在必填项未完成前不可点击；当前仅生日必填，性别、时辰与地区允许留空，时辰和地区也支持手动清空。  
`InputPage.tsx` 当前首页仅保留系统简介、信息录入与服务说明三块内容，顶部功能点压缩为 2x2 简介能力卡片，并在录入区上方以内联提示承载地区与提交状态；提交前会为缺失档案名补齐默认命名，并把未选择性别转成“未知”口径发送给后端。  
`RecordsPage.tsx` 负责档案管理、单框搜索、紧凑列表布局、详情弹层、编辑和删除流程，列表项采用左侧信息区与右侧固定操作区的双栏结构，并按子时/闰月条件显示特殊案例标签；缺失性别、时辰等展示统一收口为“未知”。  
`ResultPage.tsx` 当前采用左侧复合切换控制区与九宫格、右侧满高分析概览的左右布局，底部中央额外提供 AI 问答系统悬浮入口，套数切换已并入阴阳格切换区域。  
`RecordDetailDialog.tsx` 负责档案管理内的只读详情弹层，现已对齐结果页的控制区实现，采用左侧复合切换控制区与九宫格、右侧满高分析概览的紧凑左右分栏布局。  
`lib/numerologyProfile.ts` 负责把主魂与九宫格串派生成图腾、天赋和特质，供结果页与详情页共享灵魂结构扩展信息。  
`lib/shichen.ts` 负责统一把单字时辰补齐成“xx时”，并集中处理“未知”口径，供结果页、详情页和档案列表复用。
