# 仓库维护规范

任何功能、架构、写法更新，必须在工作结束后同步更新相关目录的 `ARCHITECTURE.md` 与被改源码文件的开头注释。  
本文件定义全仓库文档维护规则、目录地图和统一模板。  
详细结构说明下沉到各主源码目录自己的 `ARCHITECTURE.md`。

## 主要目录地图

- `backend`：HTTP API、鉴权、持久化与服务编排
- `frontend`：Web 端页面、组件、状态与接口调用
- `nine_grid`：独立算法包与命令行入口
- `docs`：业务与算法说明文档
- `tests`：独立算法与地区数据测试

## 目录文档模板

- 开头第一行固定写：`一旦我所属的文件夹有所变化请更新我`
- 三行以内说明目录职责
- 下方列出本目录直接子文件：
  - 文件名
  - 地位
  - 功能

## 文件头注释模板

源码文件开头统一写四行极简注释：

```text
input: 依赖外部什么
output: 对外提供什么
pos: 在系统局部的地位是什么
一旦我被更新务必更新我的开头注释以及所属文件夹的 md
```

## 覆盖范围

- 目录级文档只覆盖：根目录、`backend`、`backend/app`、`frontend`、`frontend/src`、`nine_grid`、`docs`、`tests`
- 文件头注释只覆盖项目自写源码：
  - `backend/app/**/*.py`
  - `frontend/src/**/*.{ts,tsx}`
  - `nine_grid/**/*.py`
- 不覆盖 lockfile、构建产物、缓存、图片、数据库文件、测试文件和配置文件

## 根目录文件

| 文件名 | 地位 | 功能 |
| --- | --- | --- |
| `README.md` | 对外入口 | 说明项目用途与基本使用方式 |
| `.env.example` | 本地配置模板 | 提供邮件等环境变量示例 |
| `docker-compose.yml` | 运行编排入口 | 定义前后端稳定运行所需容器与卷 |
| `.dockerignore` | Docker 构建配置 | 排除无关构建上下文与本地产物 |
| `region.json` | 根级静态数据 | 提供地区树形原始数据 |
| `LICENSE` | 法律说明 | 声明仓库授权方式 |
| `.gitignore` | 工程配置 | 约束不纳入版本控制的文件 |
